Notes on the Flash Slide show

The flash slideshow uses an in house developed system. You are allowed to use this flash app but it will not be supported by Villarenters.

The Flash application gets its data from an xml file that contains a list of images. This file uses the images.asp file in the slideshow folder.

The xml follows the format below

<?xml version="1.0" encoding="UTF-8"?>
<gallery>
<album id='Property 1933' description='Property Advert Pictures' lgPath='' startHere='true'>
<img src='http://www.rentalsystems.com/data/images/1933/1933_1.jpg'  caption=' '/>
<img src='http://www.rentalsystems.com/data/images/1933/1933_2.jpg'  caption=' '/>
<img src='http://www.rentalsystems.com/data/images/1933/1933_3.jpg'  caption=' '/>
<img src='http://www.rentalsystems.com/data/images/1933/1933_4.jpg'  caption=' '/>
</album>
</gallery>

The xml service is used by the images.asp to create the correctly formatted XML

 

 

 

Notes on accessing XML using the Microsoft DOM object

 

This is the method I use to select the values from an xml stream.There are other way to access xml data but I prefer this.

 

The code below is looking for the villa element (this contains the data for each villa)

The (0) indicates that the first villa result should be used. For example we would use this in the villa details page where there is only one villa anyhow,

xml.getElementsByTagName("Villa")(0).childNodes(3).firstChild.nodevalue

 

If I wanted to access multiple villas the code would need to be added into a loop. This would be handy for the results page.

 

For intcount=0 to 100

xml.getElementsByTagName("Villa")( intcount).childNodes(3).firstChild.nodevalue

next

 

The second part of the code the .childNodes(3).firstChild.nodevalue accesses node number threes value.

 

There is a more friendly way of doing things and I have used it now and then such as.

xml.getElementsByTagName("Title")(3).firstChild.nodeValue which accesses the Title of the third result directly.
